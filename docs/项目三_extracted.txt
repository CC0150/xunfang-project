一、产品族管理模块-SRS
A1. 背景与目标
背景：产品主数据需要在“产品族（Family）→ 产品（Product）”两级结构下统一管理。产品族用于归类产品、便于搜索与后续派生。目标：提供产品族主数据的查询、分页、查看、创建、更新、删除（批量）等能力；前端采用筛选查询 + 列表 + 编辑弹窗交互；后端打通 iDME 的 XfProductFamily 实体，支持 Like 检索与分页。
A2. 原型范围与边界
范围内：产品族 CRUD；按“中文名称/英文名称”模糊查询；分页；批量删除；基础字段校验；编码展示（前端以 PF{id} 规则派生显示）。
原型预览查询产品族:添加产品族:修改产品族:
A3. 业务流程
1.查询与分页：输入中文/英文名称 → 点击搜索 → 分页返回列表；重置恢复默认。2.新增：打开弹窗 → 填写中/英文名与描述 → 提交 → 成功后刷新列表。3.修改：行内“修改”→ 加载详情 → 保存变更。4.删除：支持单选/多选删除；确认后调用批量删除接口。

A5. 后端接口规格
列表：GET /manufacture/family/list 参数：pageNum,pageSize,productFamilyNameCn(Like),productFamilyNameEn(Like) 响应：{ rows:[XfProductFamily], total }
详情：GET /manufacture/family/{id} → { data: XfProductFamily }

A6. 前端交互与校验（Element Plus）
查询区：中文名称、英文名称；回车触发搜索；支持重置。

校验：必填项空校验；长度限制（前端与后端双侧限制）。
批量操作：选中多行后启用“删除”；无选择时置灰。
A7. 关键业务规则与异常处理
删除前确认二次弹窗；发生后端失败（如 iDME 返回错误）时，原样提示错误信息。
若将来引入“被产品引用的产品族不可删”，需在后端落拦截（当前代码未强制约束，建议）。
二、产品管理模块-SRS
B1. 背景与目标
背景：产品为“可检入/检出”的版本化对象（Version Object），存在工作状态与生命周期状态两类状态维度，且支持附件（文件）归档。目标：实现产品查询、分页、查看、创建（含选择生命周期模板/目标状态）、修改（需先检出）、检入/检出、生命周期状态更新、删除（含批量，受状态限制）、附件上传/下载等能力；前端配套远程下拉选择产品族。
B2. 原型范围与边界
范围内：查询（名称/产品族/类别）、分页、详情、新增（必选生命周期信息）、修改（仅检出后）、检出、检入、更新生命周期状态、行删/批删（受状态限制）、附件上传/下载。

B3. 业务流程（要点）
1.查询与分页：按产品名称（like）、产品族（like）、类别（=），只返回最新版本（latest=true），按更新时间倒序。2.新增：必填：产品名称、产品族、类别；必须选择：生命周期模板 → 业务操作（默认 create）→ 目标生命周期状态；可选：规格、描述、附件（单文件）。3.修改：前置条件：工作状态=已检出（UI 统一语义 CHECKED_OUT）。允许更新基础属性与附件；支持“替换/清除/保留”三种附件策略。4.检出：基于 masterId，带上 lifecycleTemplate/state；workCopyType 默认 BOTH。5.检入：基于 masterId，带上 lifecycleTemplate/state；viewNo 可传空串。6.更新生命周期状态：操作固定 edit，先选业务操作，再选目标状态，提交后刷新。7.删除：单删允许在“已检入”状态下进行；批删：若选择集中包含“已检出”记录则禁用。8.附件下载：从 extAttrs 解析 fileId/name，拼装后端下载 URL 并流式下载。

B5. 后端接口规格（产品）
列表：GET /manufacture/product/list 参数：pageNum,pageSize,productName(Like),productFamily(Like),category(=)；只返回 latest=true 响应：{ rows:[XfVersionProduct], total }（已提取附件名与下载地址）
详情：GET /manufacture/product/{id} → { data: XfVersionProduct }
新增：POST /manufacture/product（multipart/form-data；两段式：data JSON + 可选 file）
修改：PUT /manufacture/product（与新增相同表单结构；新增 id；支持附件替换/清除/保留：clearFile=true）
单删（传 masterId）：DELETE /manufacture/product/deleteXfProductByProductId/{id}
批删（传 masterId 串）：DELETE /manufacture/product/{ids}
检出：PUT /manufacture/product/checkout 样例：{ masterId, workCopyType:’BOTH’, lifecycleTemplate:{id}, lifecycleState:{id} }
检入：PUT /manufacture/product/checkin 样例：{ viewNo:’’, masterId, lifecycleTemplate:{id}, lifecycleState:{id} }
更新生命周期状态：PUT /manufacture/product/updateStatus 样例：{ id, workingState:’INWORK’|’CHECKED_IN’, lifecycleTemplate:{id}, lifecycleState:{id} }
B6. 生命周期相关接口（前端依赖）
模板列表：GET /lifecycle/lifecycleTemplateList（支持 master.businessCode 与 latest 检索）
业务操作（单个/列表）：GET /lifecycle/lifeBusiness?templateId=…&operation=create|edit&stateId=?，GET /lifecycle/lifeBusinessList?…
目标状态：GET /lifecycle/lifeState?templateId=…&businessOperationId=…&operation=create|edit&stateId=?新增页默认选择 operation=create；“更新状态”页固定 operation=edit，强制用户重新选择业务操作与目标状态。
B7. 前端交互与校验（Element Plus）
查询区：产品名称、产品族（远程模糊）、类别（CU/ST 字典）；支持回车搜索与重置。
表格列：锁图标列（uiWorkingState===’CHECKED_OUT’ 时显示红色锁）；编码（PO{id}）、名称、产品族、类别（字典转中文）、版本、规格、生命周期状态（彩色 Tag）、附件名（可点下载）、描述、创建日期、操作。
操作按钮：工作状态=已检入：显示“检出”“删除”；工作状态=已检出：显示“修改”“检入”；“更新状态”始终可见。批量删除：若选择集中含“已检出”记录则禁用。
新增/修改弹窗：新增时必选“生命周期模板→业务操作（默认 create）→目标状态”；附件上传：仅 1 个文件，大小 ≤ 5MB；允许扩展名：png/jpg/jpeg/gif/bmp/webp/pdf/doc/docx/xls/xlsx/ppt/pptx/txt；类别：CU/ST 字典；产品族：远程下拉。
下载：点击附件名通过流式下载。
B8. 关键业务规则
工作状态统一映射：INWORK（iDME“工作中”）在前端一律视为 CHECKED_OUT；CHECKED_IN 为已检入。
修改前置：仅 CHECKED_OUT 可修改；尝试在检入态修改时提示“请先检出，再进行修改”。
删除限制：CHECKED_OUT 不可删除；批删若包含检出记录则禁用并提示。
生命周期绑定：新增/检出/检入/更新状态均需带 lifecycleTemplate.id / lifecycleState.id。
附件策略：修改时三选一——保持不变 / 替换新文件 / 清除原文件（clearFile=true）。
仅返回最新版本：列表查询默认 latest=true。
文件名/下载地址：从 extAttrs[name=File] 的首个条目解析 fileId/name 并拼接后端下载 URL。
B9. 非功能与安全
分页性能：后端统一 startPage 拦截分页；前端每页默认 10 条。
输入校验：前后端双检（必填、长度、文件大小/后缀）。
异常透明：后端将 iDME 错误透传至前端，便于排障。
三、跨模块联动——产品 × 产品族 × 生命周期 × 文件
联动点
说明
产品 ←→ 产品族

四、典型测试用例
9.列表只显最新版本：创建同一产品的多个版本，查询结果仅出现最新一条。10.新增必选生命周期：不选择模板/业务操作/目标状态直接提交 → 前端/后端拦截。11.检出后修改：检入态点击“修改”→ 提示“请先检出”；检出后可编辑并保存成功。12.更新状态（edit）：打开“更新状态”→ 必须手选业务操作与目标状态 → 成功后列表状态变更。13.删除保护：选择含检出记录进行批删 → 按钮置灰或提示不可删除。14.附件三策略：①不更换文件保持原样；②替换为新文件后列表展示新文件名；③选择“移除文件”后不再展示下载链接。15.远程族下拉：输入关键字能模糊命中中文名；清空关键字可加载默认一页族列表。16.类别字典：保存 CU/ST，表格显示“定制/标准”。17.文件校验：上传超 5MB 或非法后缀 → 前端立即拦截并提示。

import request from '@/utils/request'

1.export function listFamily(query)：定义并导出一个名为 listFamily的函数，接收一个 query 参数，query 是包含查询条件（如分页参数、筛选条件等）的对象。2.return request({…})：调用封装的 request 方法发送请求，并返回请求的 Promise（方便后续用 async/await 或 .then() 处理响应）。(1)url: ‘/manufacture/family/list’：指定请求的后端接口路径。(2)method: ‘get’：设置请求方法为 GET。(3)params: query：将 query 对象中的键值对作为 URL 的查询参数其他函数以此类推
// 查询产品族详细

  return request({
    url: '/manufacture/family',

    data: data
  })
}
// 单个删除产品族
export function delFamily(id) {
  return request({
    url: '/manufacture/family/delete/' + id,
    method: 'delete'
  })
}
// 批量删除产品族（
export function batchDelFamily(ids) {
  if (!Array.isArray(ids)) ids = [ids]
  const safeIds = ids.map(i => (i === null || i === undefined ? '' : String(i).trim())).filter(Boolean)
  if (safeIds.length === 0) {
    return Promise.reject(new Error('没有可删除的 id'))
  }
  return request({
    url: '/manufacture/family/' + safeIds.join(','),
    method: 'delete'
  })
}

function extractId(row) {
  if (!row || typeof row !== 'object') return null
  const cand = row.id ?? row.productFamilyId ?? row.productFamilyCode ?? null
  if (cand === null || cand === undefined) return null
  const s = String(cand).trim()
  return s.length ? s : null
}

  loading.value = true
  listFamily(queryParams.value)
    .then(res => {
      const rows = res.rows || []

        ...item,
        id: extractId(item) ?? String(item.id ?? '')
      }))
      total.value = res.total || 0
      ids.value = []
      single.value = true
      multiple.value = true
      console.log('[DEBUG] getList -> ids sample:', familyList.value.map(r => r.id))
    })
    .catch(err => console.error('getList error', err))
    .finally(() => (loading.value = false))
}

/** 统一工具：有文件或已是 FormData 就走 multipart；否则走 JSON */
function sendWithOptionalFile({ url, method, data, file }) {
  // 1) 已经是 FormData：直接发
  if (isFormData(data)) {
    return request({ url, method, data })
  }
  // 2) 传了原始文件：拼 FormData
  if (file) {
    const fd = new FormData()
    fd.append('data', new Blob([JSON.stringify(data || {})], { type: 'application/json' }))
    fd.append('file', file, file.name)
    return request({ url, method, data: fd })
  }
  // 3) 纯 JSON
  return request({ url, method, data })
}

const parseMaybeJson = (val) => {
  if (typeof val === "string") {
    const s = val.trim()
    if ((s.startsWith("{") && s.endsWith("}")) || (s.startsWith("[") && s.endsWith("]"))) {
      try { return JSON.parse(s) } catch (e) {}
    }
  }
  return val
}
const codeOfCategory = (val) => {
  const v = parseMaybeJson(val)
  if (!v) return null
  if (typeof v === "string") return v
  return v.alias || v.code || v.value || null
}

  if (v && typeof v === "object") {

    return hit ? hit.label : (v.cnName || v.enName || code || "-")

}

  const ws = row?.workingState

  const tplId = flowDlg.form.lifecycleTemplateId
  const op = flowDlg.form.operation

  if (!tplId || !op) return
  flowDlg.loading.business = true
  try {
    const bizRes = await getLifecycleBusiness({ templateId: tplId, operation: op, stateId: curStateId })
    const list = Array.isArray(bizRes?.data) ? bizRes.data : (bizRes?.data ? [bizRes.data] : [])
    flowDlg.bizOptions = list
    let picked = null, targets = []
    for (const bo of list) {
      const r = await getLifecycleStates({
        templateId: tplId, businessOperationId: bo.id, stateId: curStateId, operation: op
      })
      const arr = r?.data || r?.rows || []
      if (Array.isArray(arr) && arr.length) { picked = bo; targets = arr; break }
    }
    if (!picked) { flowDlg.stateOptions = []; flowDlg.form.businessOperationId=''; flowDlg.form.lifecycleStateId=''; return }
    flowDlg.form.businessOperationId = picked.id
    flowDlg.stateOptions = targets
    flowDlg.form.lifecycleStateId = targets[0]?.id || ''

返回：TableDataInfo（rows=data, total=pageInfo.totalRows）。回到Controller类，注入3个注解，同时声明服务层接口的成员变量，后续通过依赖注入获取服务层的实现对象，从而调用服务层的业务方法：以下是对3个注解的解释：（1）@RestController是 Spring 框架的注解，标记当前类为 “REST 风格的控制器”。它是 @Controller（声明类为 “控制器”，能接收 HTTP 请求）和 @ResponseBody（让方法返回值直接转为 JSON 等格式响应给前端，而非跳转页面）的 “组合注解”。加了 @RestController 后，这个类里的方法不用额外加 @ResponseBody，返回的对象会自动序列化为 JSON 给前端。（2）@RequestMapping(“/unit”)是 Spring 框架的注解，指定当前控制器处理的 “请求路径前缀”。比如这里写 @RequestMapping(“/family”)，那么这个控制器下所有接口的路径，都会以 /family开头。示例中 @GetMapping(“/list”) 的完整路径就是 http://xxx.com/family/list，方便对相关接口做路径分组。（3）@Resource是 Java 标准的注解，用于依赖注入。把 “服务层接口 IXfProductFamilyService的实现类” 注入到控制器里，这样控制器就能调用 xfProductFamilyService.selectXfProductFamilyById(…) 等服务层方法，实现 “控制层 → 服务层” 的调用。以下是后续编写控制类时需要用到的注解：

/**
   * 查询产品信息
   * 
   * @param id 产品信息主键
   * @return 产品信息
   */
  @Override
  public XfVersionProduct selectXfProductById(String id) {
      String token = RedisUtils.get(DMEUtil.tokenKey);
      String url = DMEUtil.projectUrl + DMEUtil.executeApiUrl+"/XfProduct01/get";
      // 构造请求
      JSONObject root = new JSONObject();
      JSONObject params = new JSONObject();
      params.put("id", id);
      root.put("params", params);
      // 请求
      String res = RequestUtil.requestsPost(url, root.toString(), token);
      JSONObject jo = JSONObject.parseObject(res);
      // 结果校验
      if (!"SUCCESS".equalsIgnoreCase(jo.getString("result"))) {
          throw new RuntimeException("查询失败: " + jo.getJSONArray("errors"));
      }
      JSONArray data = jo.getJSONArray("data");
      if (data == null || data.isEmpty()) {
          return null;
      }
      JSONObject p = data.getJSONObject(0);
      // 映射到 XfPartVersion（XfPart 的所有字段 + masterId / creator / modifier）
      XfVersionProduct product = new XfVersionProduct();
      product.setId(p.getString("id"));                     // 这是“版本对象 id”，用于更新 data.id
      product.setProductName(p.getString("productName"));
      product.setProductFamily(p.getString("productFamily"));
      product.setSpecificationModels(p.getString("specificationModels"));
      product.setProductDescribe(p.getString("productDescribe"));
      // 仅取枚举中的 alias  产品类别
      product.setCategory(dmeClient.pickAlias(p.get("category")));
      // 工作状态透传 & 派生 uiWorkingState（方法内部已处理对象/字符串）
      product.setWorkingStateFlex(p.get("workingState"));
      // masterId（检出并更新会用到）
      JSONObject master = p.getJSONObject("master");
      if (master != null) {
          product.setMasterId(master.getString("id"));
      }
      // 带上创建/修改人
      product.setCreator(p.getString("creator"));
      product.setModifier(p.getString("modifier"));
      String createTime = p.getString("createTime");
      if (createTime != null) {
          try {
              product.setCreateTime(DMEUtil.dmeDateToExamDate(createTime));
          } catch (Exception ignore) {}
      }
      JSONArray extAttrs = p.getJSONArray("extAttrs");
      if (extAttrs != null && !extAttrs.isEmpty()) {
          @SuppressWarnings("unchecked")
          List<Map<String, Object>> list =
                  (List<Map<String, Object>>) (List<?>) extAttrs.toJavaList(Map.class);
          product.setExtAttrs(list);
          product.extractFileName(); // → 会自动填充 fileId / fileName / fileNameNoExt / fileDownloadUrl
      }
      return product;
  }

          params.put("extAttrs", extAttrs);
      } else if (wantClear) {
          // B. 删除原文件（未上传新文件）：extAttrs = [{ name:"File", type:"FILE", value:[] }]

          fileAttr.put("name", "File");

      paramsJson.put("params", params);
      String res = RequestUtil.requestsPost(url, paramsJson.toString(), token);
      Object result = JSONObject.parseObject(res).get("result");

private static void putIfHasId(JSONObject target, String key, Map<String, Object> srcMap, String fallbackId) {
      String id = null;
      if (srcMap != null) {
          Object v = srcMap.get("id");
          if (v != null) id = String.valueOf(v);
      }
      if (id == null && fallbackId != null && !fallbackId.trim().isEmpty()) {
          id = fallbackId;
      }
      if (id != null && !id.trim().isEmpty()) {
          JSONObject o = new JSONObject();
          o.put("id", id);
          target.put(key, o);
      }
  }
  private static String nvl(String v, String def) {
      return (v == null || v.trim().isEmpty()) ? def : v;
  }

      final String path = isInWork
              ? "/XfProduct01/update"

 /**
     * 查询生命周期模板列表
     * @param lifecycleTemplate
     * @return
     */
    @Override
    public TableDataInfo selectLifecycleTemplateList(LifecycleTemplate lifecycleTemplate) throws Exception {
        String url = DMEUtil.commonApiUrl+"/LifecycleTemplate/find/20/1";
        // === 手工拼装 IDME 所需结构 ===
        JSONObject root = new JSONObject();
        JSONObject params = new JSONObject();
        JSONObject filter = new JSONObject();
        JSONArray conditions = new JSONArray();
        filter.put("joiner", "and");
        // master.businessCode like
        if (lifecycleTemplate.getMaster() != null) {
            String bc = lifecycleTemplate.getMaster().getBusinessCode();
            if (bc != null && !bc.isEmpty()) {
                JSONObject c1 = new JSONObject();
                c1.put("conditionName", "master.businessCode");
                c1.put("operator", "like");
                // 注意：这是字符串数组
                c1.put("conditionValues", Collections.singletonList(bc));
                conditions.add(c1);
            }
        }
        // latest =  —— 注意：这里放布尔值，而不是字符串
        if (lifecycleTemplate.getLatest() != null) {
            JSONObject c2 = new JSONObject();
            c2.put("conditionName", "latest");
            c2.put("operator", "=");
            // 直接放 Boolean，保证序列化为 true/false
            c2.put("conditionValues", Collections.singletonList(lifecycleTemplate.getLatest()));
            conditions.add(c2);
        }
        filter.put("conditions", conditions);
        params.put("filter", filter);
        params.put("isNeedTotal", true);
        root.put("params", params);
        // === 手工拼装结束 ===
        DMEResponse<LifecycleTemplate> response = dmeClient.post(
                url, root, new TypeReference<DMEResponse<LifecycleTemplate>>() {}
        );
        return TableDataUtils.buildTableData(response);
    }
